# 认定管理类文件模式

## 适用对象

用于政府部门对某类政策载体进行认定、评定、认证、建设管理、绩效评价的文件。常见对象包括：

- 中心：创新中心、技术中心、服务中心、研发中心、适配中心。
- 平台：公共服务平台、创新平台、检验检测平台。
- 基地和园区：示范基地、产业基地、园区、试点园区。
- 实验室和单位：重点实验室、示范单位、试点单位。

## 常见组合

完整文件通常包括：

1. 红头通知。
2. 管理办法正文。
3. 申报、评价或证明类附件。

也可以按用户需求只生成其中一类，如申报指南、绩效评价办法、年度考核通知、附件模板。

## 红头通知

通常包含：

- 发文机关红头。
- 发文字号。
- 红色分隔线。
- 通知标题。
- 主送单位。
- 印发说明正文。
- 发文机关和日期落款。

事实不明时使用占位符，不虚构文号、日期、依据或部门决定。

## 管理办法正文

章节按主题取舍，不机械套用。常用章节包括：

- 总则：制定目的、适用范围、对象定义、主管部门职责。
- 条件要求：主体资格、能力条件、信用合规、场地团队、设备或服务能力。
- 申报与认定：申报通知、材料提交、形式审查、专家评审、现场核查、公示、认定。
- 申报资料：申报书、建设方案、证照材料、证明材料、信用记录、团队清单、联合申报协议。
- 评估及管理：年度报告、绩效评价、整改、警示、撤销、重大变更报告。
- 扶持措施或政策措施：授牌、宣传、场景或项目支持、生态对接、符合条件的资金支持。
- 附则：解释机关、施行时间、有效期或上位规定衔接。

可按需要增加保障机制、监督管理、运行机制、绩效评价、退出机制、职责分工等章节。

## 附件材料

常见附件包括：

- 认定申报书：申报单位信息、联合申报单位、业务范围、联系人、基本情况。
- 建设或实施方案：建设基础、单位概况、方向任务、目标计划、组织保障。
- 能力条件申报表：能力指标、支撑材料、申报单位说明。
- 证明材料清单或证明模板：证明主体、证明事项、盖章、日期。
- 承诺书：材料真实性、接受监督、持续符合条件、重大变更报告、责任承担。

用户需要更完整材料包时，可增加专家评审表、绩效评价表、材料核验表。

## 写作语气

- 使用克制、正式的政策语言。
- 避免宣传口号、营销词和无依据承诺。
- 条款编号保持连续，正文主结构使用 `第X章`、`第X条`。
- 条款内列举材料、条件、情形时使用 `（一）`、`（二）`。
- 章节标题应短，且贴合主管部门、认定对象和政策目的。

## PDF 版式提示

- A4 竖版。
- 首页包含红头、文号、红线、通知标题、主送单位、正文和落款。
- 正文使用正式公文排版，正文常规字重，标题和章节标题较重。
- 页码稳定，不压正文、表格或版记。
- 附件表格边框清楚，行高适中。
- 右侧填写项需保留填写空间，尤其是 `申报单位盖章：`、`填表人：`、`联系电话：`、`填表日期：`。
- 末页版记为上下横线，左侧印发机关，右侧印发日期。
